# Prisoners — Дилемма заключённого (8-bit симуляция)

## Проект
Веб-страница с 8-битной графикой (стиль NES/Dendy), симулирующая решение Дилеммы заключённого.

**Что видит пользователь:**
- Краткое описание задачи (Prisoner's Dilemma и что это симуляция)
- Выбор языка (RU/EN) — кнопки с флагами: **🇷🇺 флаг России** и **🇺🇸 флаг США**
- Кнопка «Старт» → выбор стратегии с кратким описанием
- Анимация процесса → итоговая статистика

---

## Задачи

### 1. Разработка index.html

#### 1.1 Каркас страницы
- [ ] HTML5 + inline `<style>` + inline `<script>` — один файл
- [ ] Пиксельный шрифт через CDN (Press Start 2P)
- [ ] Canvas с CSS-масштабированием, `image-rendering: pixelated`
- [ ] **Разрешение канваса:** 512×360 (увеличенное против 256×240) — каждый «пиксель» спрайта должен быть читабельным на современных экранах: номера коробок, номера заключённых должны явно различаться
- [ ] Цветовая гамма: классический NES (светлый фон, яркие цвета)

#### 1.2 Локализация
- [ ] JS-объект `i18n` с ключами `ru` / `en`
- [ ] Все тексты (описание, кнопки, подписи, статистика) через объект
- [ ] **Селектор языка:** две кнопки с изображением флагов: флаг России (🇷🇺), флаг США (🇺🇸)
- [ ] Активная кнопка языка — яркая подсветка / рамка

#### 1.3 Логика симуляции
- [ ] 100 заключённых, 100 коробок
- [ ] Каждому заключённому — до 50 попыток (N/2)
- [ ] **Стратегия №1 — Рандом:** заключённый хаотично выбирает коробки, не запоминая результаты
- [ ] **Стратегия №2 — Циклы:** заключённый начинает с коробки под своим номером, затем переходит к коробке, номер которой лежит в текущей
- [ ] Один прогон: все 100 человек проверяют коробки. Кнопка «Старт» запускает повторно.

#### 1.4 Анимация (режим: подробно, каждого по шагам)
- [ ] **Пиксельные спрайты — заключённый (стиль Mario, 16×24 px):**
  - Голова (телесный цвет, круглый контур)
  - Глаза (два пикселя на лице)
  - Туловище — **полосатая роба** (чередование горизонтальных белых и чёрных/тёмных полос)
  - Номер заключённого (белый текст на тёмном прямоугольнике на груди)
  - Руки (по бокам, телесного цвета) — анимируются при ходьбе
  - Ноги (телесного цвета, две ноги, двигаются при ходьбе)
  - Персонаж повёрнут анфас или в профиль (в зависимости от направления движения)
- [ ] **Коробки:** деревянные текстурированные ящики с чётким номером на боку
- [ ] **Тюрьма — три комнаты + внешняя территория:**

  **Текстура стен:**
  - Снаружи тюрьмы: **красный кирпич** (текстура из пикселей — чередование кирпичного узора)
  - Изнутри тюрьмы: **серый бетон** (гладкая серая заливка) — стены плотно прилегают друг к другу (один пиксель стен снаружи = кирпич, следующий же пиксель внутри = серый)

  **Комнаты (слева направо):**
  1. **Комната заключённых (small)** — стоят все 100 заключённых (ожидают на местах)
  2. **Комната с коробками (большая)** — ряды коробок с номерами (10×10)
  3. **Внешняя территория тюрьмы:**
     - **Пол:** зелёная трава (пиксельная текстура травы)
     - **Ограждение:** забор/сетка по краю территории
     - **За забором:** продолжение травы, зелёный фон

  **Пол внутри тюрьмы:**
  - Серый с чёрными точками (текстура «асфальт/бетон»)

  **Фон вокруг тюрьмы:**
  - Зелёная трава вокруг всего периметра
  - Сверху и снизу экрана — **синие полосы** (речка/вода, синяя текстура с волнистым узором)

- [ ] **Анимация передвижения:** заключённый **заходит справа** в свою комнату → подходит к первой выбранной коробке
- [ ] **Открытие коробки:** анимация крышки (поднимается), видно номер внутри
- [ ] **Перемещение** к следующей коробке (пошагово) — человечек идёт по «асфальту»
- [ ] **Анимация успеха:** если заключённый нашёл свой номер — **зелёный восклицательный знак (!)** над головой, с анимацией появления (pop-in)
- [ ] **Анимация провала:** если не нашёл за 50 попыток — **красный крест (✕)** над головой
- [ ] После завершения — заключённый уходит влево/в правую колонну
- [ ] `requestAnimationFrame`, покадровая смена спрайтов
- [ ] Чекбокс «Показывать текст»/«Show text» — подписи хода (какую коробку открыл, какой номер нашёл)

#### 1.5 Экран статистики
- [ ] После завершения прогона — итоговый экран
- [ ] Сколько заключённых нашли свои номера (успех) / не нашли (провал)
- [ ] Процент успеха
- [ ] Для стратегии циклов ожидаем ~31% побед на 100 прогонах (пояснение: при 100 прогонах * 100 человек = 10000 попыток, ~31% успеха у стратегии циклов)
- [ ] Показывается для текущего (одного) прогона

### 2. Поток пользователя

1. Открытие страницы → описание дилеммы + кнопки выбора языка (🇷🇺/🇺🇸)
2. Кнопка «Старт» → выбор стратегии (с кратким описанием каждой)
3. Анимация: все 100 заключённых по очереди проверяют коробки
4. Экран со статистикой (успех/провал, проценты)
5. Возможность нажать «Старт» снова

## Технические требования

- HTML5 + CSS3 + Vanilla JS (ES6+)
- Canvas 2D API для отрисовки
- Никаких зависимостей — только CDN-шрифт
- Открывается из коробки: `open index.html`
